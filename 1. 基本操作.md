# 基本操作

## 权限模型

CC有三种权限模式：

- Plan mode：先生成思路，用户同意后再运行
- Ask before edits：每次修改代码前都要询问用户许可
- Edit automatically：自动修改代码，但进行系统操作时依然需要询问用户同意

以上三种权限模式可以通过Shift+Tab切换

如果想全自动流程操作，可以设置为dangerously-skip-permissions，并让Claude Code将一些常用系统操作加入白名单，就可以大幅提升自动化水平

**Effort**是努力程度，用于设置CC再任务上投入深度思考的程度，它决定了模型在回答前花多少时间“深思熟虑”。

三个级别：

- Low：快读响应，但复杂推理可能不够
- Medium：默认级别，在速度和深度之间取得平衡
- High：思考时间更长，推理深度适合复杂任务

### 2. @文件

通过@具体的文件，让CC明确任务文件，而不需要在整个项目目录中搜寻，提高效率节省token。

另一种使用@的场景是，当提示词非常长的时候，可以先在本地建立一个md文档，然后再在cc中@该文件，让cc按照文档中的提示词去运行。

**冷知识**：提供的提示词越长，命令越明确，反而越省token。

如果采用的是多模态模型，则理论上还可以看图片，但是deepseek不太行。

### 3. 斜杠命令

- /help: 关于斜杠命令的帮助说明
- /modal: 切换模型
- /btw: by the way，用于询问一些与正在进行的项目无关的内容，该对话与原本的上下文内容是隔离开的，聊完后按esc即可退出，回到原来的项目上下文中
- /simplify: 启动3个审查Agent，从代码质量、运行效率和是否可复用的角度进行代码审核，并自动进行优化修改
