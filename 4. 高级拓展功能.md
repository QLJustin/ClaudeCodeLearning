# 高级拓展功能

## skill

skill与CALUDE.md的区别是，前者并不是必须用到的，但又经常用到的

- 知识型
- 流程型
- 工具型
- 混合型

skill的运行方式是，每次运行agent的时候，给llm发送几行简单的元信息，上面写着skill的名称和什么时候要用这个skill。大模型自行判断什么时候要使用这些skill

skill也分为全局级和项目级，存放的位置分别为：

全局级：%USERFROFILE%/.claude/skill/技能名/SKILL.md

项目级：项目名/.claude/skill/技能名/SKILL.md

调用方法是：如果agent认为你需要用到该skill，他会自动调用。或者我们可以直接用自然语言聊天的方式调用skill，或者手动用“/SKILL名”来调用特定的skill。因为当skill存多了之后，以及上下文存的比较满了之后，或者技能触发词条写的不够准确，都有可能无法自动触发skill。

### skill的安装

可以使用一个叫做“find skills”的skill，安装之后，直接以聊天形式询问是否有对应需求的skill，它会自动寻找合适的skill来推荐

### 自建skill

更多时候我们需要自己创建skill，可以直接使用官方的skill creator。输入需求即可生成可复用的skill文件

## MCP

MCP实际上就是agent与外部工具、外部服务连接（如其他软件）的转接头

MCP比较烧token，现在很多的外部工具，轻量的转向了skill，重量的转向了CLI

## CLI

CLI即命令行工具，意思是过去程序员们专用的一些命令行代码。

例如普通人想下载一个软件要去网页上操作半天，程序员一个命令就完成了；普通人想压缩一个视频要去找专门的软件，程序员也是一个命令就完成了。

现在对于AI，复现普通人的操作是很麻烦的，但是它是很会用CLI的，用这种方式来操作又精准又高效。

于是很多厂商就把原先需要人类点击操作的东西也做成了CLI，目的就是让AI能更容易地进行快速操作。

## SubAgent

子agent具有独立的上下文空间，且可以多个并行，这样就可以实现主agent继续写自己的代码，且不会扰乱其上下文的情况下，完成一些附属操作（不能新开页面，因为这样会丢失上下文）

当有的任务比较复杂，且agent认为可以进行并行派生的时候，会自动创建子agent

还可以手动创建，输入/agent即可

## Hook

一个自动触发器，可以设置：当cc怎么怎么样的时候，就让cc干啥啥。

可以直接通过自然语言和他说：`给我做一个hook，实现...`

## plugin

插件实际上就是把一些skill、MCP、CLI、Hook、SubAgent之类的东西打包起来。

输入/plugins调用
